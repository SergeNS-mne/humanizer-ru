Claude Code skill, который убирает признаки AI-генерации из **русскоязычных** текстов.

Ловит канцелярит, цепочки родительного падежа, «является»-паразит, кальки с английского, штампы «современной эпохи», избыточные преувеличения, правило трёх и типографические AI-маркеры.

## Зачем это

Современные LLM (Claude, GPT, Gemini) на русском пишут узнаваемо: длинные придаточные, номинализация, «в современном мире, где...», «не просто X, это Y», «является неотъемлемой частью», цепочки из трёх абстрактных существительных подряд. Это читается как AI, даже если фактология безупречна.

`humanizer-ru` — каталог из ~38 паттернов и правил переписывания, которые убирают эти маркеры и дают более живой текст.

Работает в двух режимах:
- **Базовый** — просто чистит текст по правилам.
- **Voice calibration** — анализирует ваш образец письма (2–3 абзаца) и подстраивается под вашу манеру: «ты» или «Вы», тип тире, «ёлочки» или прямые кавычки, использование ё, ритм предложений.

## Установка

### Claude Code

```bash
mkdir -p ~/.claude/skills
git clone https://github.com/SergeNS-mne/humanizer-ru.git ~/.claude/skills/humanizer-ru
```

Или вручную:
```bash
mkdir -p ~/.claude/skills/humanizer-ru
curl -o ~/.claude/skills/humanizer-ru/SKILL.md \
  https://raw.githubusercontent.com/SergeNS-mne/humanizer-ru/main/SKILL.md
```

### OpenCode

```bash
mkdir -p ~/.config/opencode/skills
git clone https://github.com/SergeNS-mne/humanizer-ru.git ~/.config/opencode/skills/humanizer-ru
```

> OpenCode также сканирует `~/.claude/skills/` — одного клона достаточно для обоих клиентов.

## Использование

### Базовое

```
/humanizer-ru

[вставьте текст]
```

### С калибровкой под ваш стиль

```
/humanizer-ru

Образец моего письма:
[2–3 абзаца, написанных лично вами]

Теперь очисти этот текст:
[AI-текст, который нужно переписать]
```

Скилл проанализирует ваш ритм, выбор слов, типографические привычки — и применит их в переписывании, вместо того чтобы выдавать «стерильно-чистый» текст.

## Что умеет — кратко

| Категория | Примеры паттернов |
|---|---|
| **Канцелярит** | «осуществление проверки» → «проверка» |
| **Родительный падеж** | «улучшение качества обслуживания клиентов компании» → «клиенты стали довольнее» |
| **«Является»** | «является эффективным» → «эффективен» |
| **Штампы эпохи** | «в современном мире, где...» → убрать |
| **Кальки с английского** | «в конце дня», «погрузимся», «меняет игру» |
| **Правило трёх** | «быстро, надёжно и удобно» → конкретика |
| **Отрицательные параллелизмы** | «не просто X, это Y» |
| **Преувеличения** | «революционный, уникальный, беспрецедентный» без факта |
| **Типографика** | тире, кавычки, ё — по стилю автора |
| **Чатботные артефакты** | «Надеюсь, это помогло!» |

Полный каталог — в [SKILL.md](./SKILL.md).

## Пример

**Было (AI-стиль):**
> В современном мире, где технологии не стоят на месте, AI-ассистенты являются неотъемлемой частью процесса разработки. Данные инструменты кардинально трансформируют подходы инженеров, подчёркивая их ключевую роль. Это не просто автокомплит — это раскрытие творческого потенциала в масштабе.

**Стало:**
> AI-ассистенты помогают в рутине: шаблонный код, конфиги, тесты-заготовки. Хорошо работают там, где ты всё равно не хотел бы писать руками. Опасность в том, как уверенно выглядят подсказки — легко принять код, который компилируется, но мимо задачи.
