Skill для редактирования русскоязычных LLM-текстов.

Цель — **вернуть автору голос**, а не «обмануть классификатор». Подход редакторский, в традиции [Норы Галь](https://ru.wikipedia.org/wiki/Слово_живое_и_мёртвое) и инфостиля. Опирается на классику русской стилистики, не на arms race с детекторами AI.

## Чем отличается от других хуманизаторов

Большинство скиллов работают только на уровне фраз: ловят канцелярит, штампы, кальки. Это полезно, но недостаточно. У humanizer-ru три слоя:

1. **Фразовый** — каталог из 38+ правил против AI-маркеров.
2. **Документный** — структурная диагностика (listicle-сигнатура, плотность тире, доля AI-цитат, разброс длин).
3. **Голосовой паспорт** — сохраняемый артефакт `.humanizer/voice.json` с предпочтениями автора, который **эволюционирует между сессиями**. Скилл становится тем точнее, чем чаще ты его используешь.

Плюс **жанровые пресеты** (Habr / Telegram / email / vc.ru / LinkedIn / docs) — каждый описывает ожидания канала: тип тире, формальность, разрешён ли listicle, нужны ли микро-неровности.

## Что внутри

| Слой | Делает |
|---|---|
| **Layer 1: Фразы** | Канцелярит, «является», цепочки родительного, штампы, кальки с английского, преувеличения, чатботные артефакты, типография |
| **Layer 2: Структура (audit-режим)** | Считает плотность тире, listicle-сигнатуру, долю AI-цитат, burstiness предложений и абзацев. Сообщает автору, что нужно поправить руками |
| **Layer 3: Голос (паспорт)** | Хранит обращение, тип тире, формальность, ритм, идиолект автора. Auto-evolve по образцам и feedback'у |

## Использование

### Базовое

```
/humanizer-ru

[вставь текст]
```

Если в проекте уже есть `.humanizer/voice.json` — будет использован он. Если нет — скилл предложит калибровку.

### Калибровка по образцу (рекомендуется)

```
/humanizer-ru --sample

Образец моего письма:
[2-3 абзаца, написанные лично тобой]

Текст для очистки:
[AI-текст]
```

Скилл проанализирует ритм, тире, кавычки, идиолект — сохранит в `.humanizer/voice.json` — и при следующих прогонах будет переписывать **в твоём стиле**.

### Калибровка через интервью (если образца нет)

```
/humanizer-ru --interview
```

Скилл задаст 5-7 вопросов (ты/Вы, формальность 1-10, тип тире, ирония, любимые обороты, табу) и соберёт минимальный паспорт.

### Жанровый пресет

```
/humanizer-ru --preset=habr
[текст]
```

Шесть встроенных пресетов: `habr`, `telegram`, `email`, `vc`, `linkedin`, `docs`. Каждый описывает ожидания канала по формальности, типографике, структуре.

Можно комбинировать с паспортом: `--preset=habr --sample` использует пресет как контекст канала, паспорт как личный голос.

### Audit-режим (только диагностика)

```
/humanizer-ru --audit
[текст]
```

Скилл не правит, а выдаёт отчёт по структуре: listicle-сигнатура, плотность тире, доля AI-цитат в теле, burstiness предложений и абзацев. С рекомендациями, что поправить руками.

Полезно перед публикацией на платформе с активным классификатором (Habr НЛО, vc.ru).

### Опт-ин на намеренные неровности (для соцсетей)

```
/humanizer-ru --rough
[текст]
```

В неформальных контекстах идеальная грамотность сама становится AI-маркером. Флаг `--rough` разрешает скиллу оставлять/вносить 1-2 микро-неровности (пропущенная запятая, дефис вместо тире). По умолчанию **выключено** — скилл не вносит огрехи в чужой текст без явного запроса.

### Эволюция паспорта (post-hoc feedback)

```
/humanizer-ru --reflect
[финальная версия твоей предыдущей статьи, которую ты опубликовал]
```

Скилл сравнит финал с прошлым прогоном из `.humanizer/sessions/`, поймёт, что ты откатил/изменил руками, и обновит паспорт. Через 5-10 прогонов паспорт станет точнее, чем после первой калибровки.

## Что скилл НЕ делает

Честно: **скилл не гарантирует прохождение статистических детекторов** (GPTZero, Originality.ai, Хабровский НЛО, RuBERT-классификаторы).

Скилл работает на уровне фраз и сообщает о структурных проблемах. Финальное переписывание структуры (поломать listicle, разнообразить ритм абзацев, заменить блоки AI-цитат на инлайн) требует **редакторского решения автора** — это не автоматизируется без потери смысла.

Логика: классификаторы смотрят на документ целиком (perplexity, burstiness, морфологическая равномерность). Локальные правки фраз на это влияют слабо. Скилл даёт диагностику и чек-лист, дальше работает автор.

| **Чатботные артефакты** | «Надеюсь, это помогло!» |

## Пример

**Было (AI-стиль):**
> В современном мире, где технологии не стоят на месте, AI-ассистенты являются неотъемлемой частью процесса разработки. Данные инструменты кардинально трансформируют подходы инженеров, подчёркивая их ключевую роль. Это не просто автокомплит – это раскрытие творческого потенциала в масштабе.

**Стало (пресет `habr`, тон «ты»):**
> AI-ассистенты помогают в рутине: шаблонный код, конфиги, тесты-заготовки. Хорошо работают там, где ты всё равно не хотел бы писать руками. Опасность в том, как уверенно выглядят подсказки – легко принять код, который компилируется, но мимо задачи.
